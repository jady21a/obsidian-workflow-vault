# 提示词工程入门

> [!note] 这是一篇学习笔记示例
> 学习笔记建议用「问题 → 要点 → 我的话复述 → 待办」四段式。

## 核心要点
1. **给角色**：让模型知道「你是谁」。
2. **给上下文**：背景、约束、示例（few-shot）。
3. **给格式**：明确要什么结构的输出。
4. **迭代**：一次说不清就拆步骤、让它先复述需求。

## 用我自己的话复述
提示词的本质不是「咒语」，而是「把任务讲清楚」——你对同事怎么交代，就对模型怎么说。
