# 周复盘

对**用户本人的一周**做复盘:用行为数据(每日记录 + git)对照长期目标,指出矛盾,生成结构化周复盘并落盘。

## 唯一真相 · 单一流程文件

⚠️ **本 skill 不复述流程,只指向流程。** 完整且权威的执行步骤写在:

```
5.todo/prompt/周复盘流程.md
```

被调起时:**先完整读取上面这份 `周复盘流程.md`,再严格按其中的步骤执行**(从第 0 步解析日期参数,到第 6 步跑 `update-growth.py`)。该文件内部还会要求读取两份子文件——稳定框架(`周复盘prompt-稳定.md`)和成长档案(`周复盘prompt-成长.md`),按它说的读。

> 这套复盘同时被 Claude Code(本 skill)和 Codex(`.agents.md`)使用,所以**任何流程/框架的改动都只改 `5.todo/prompt/` 下那三份文件,绝不抄进本 SKILL.md**,避免分叉。本文件只负责:① 提供触发词让自然语言("做个周复盘""复盘一下这周")也能唤起;② 把执行权交给流程文件。

## 日期参数

用户可能在触发时附带日期(如「复盘 5.14-5.21」「周复盘 5.21」)。把这段日期原样交给流程文件的第 0 步解析,不要自己另立规则。

## 不做什么

- 不在本文件里重写评估框架、输出结构、UPDATE 块格式——那些是流程文件的职责。
- 不写视频脚本(用 `video-script`)、不做事业决策(用 `career-decision`)。
- 流程要求「不询问确认,直接执行并保存」——遵守它,别在中途反复确认。
