# 周复盘 · 共享执行流程（单一真相）

> 这是 `/周复盘` 的唯一权威流程。Claude Code 的 `.claude/commands/周复盘.md` 和 Codex 的 `.agents.md` 都只是薄指针，指向本文件。**任何改动只改这里**，两个 agent 自动同步，不再分叉。

读取并严格遵守以下两份文件：

**稳定框架（评估框架、输出结构、风格要求、UPDATE 块格式）：**
`5.todo/prompt/周复盘prompt-稳定.md`

**成长档案（当前盲区清单 + 历史模式日志，作为背景上下文）：**
`5.todo/prompt/周复盘prompt-成长.md`

---

执行上下文收集（按顺序）：

**0. 解析用户指定的日期（如有）**

用户可能在"周复盘"指令后附带日期参数（Claude Code 经 `$ARGUMENTS` 传入；Codex 从用户消息中识别）。

- 如果没有给日期 → 跳过本步，按下方第 1 步确定起始日。
- 如果包含日期，则用它覆盖第 1 步，**直接使用解析结果作为起始日/截止日，不再读取 weeks 文件**。

解析规则（兼容多种格式）：
- 分隔符兼容：`.`、`-`、`/`（如 `5.21`、`5-21`、`5/21` 等价）。
- 缺省年份按当前年（2026）补全；也接受 `2026.5.21` 这类带年份的写法。
- **区间**：两个日期用 `-` 或 `~` 连接（如 `5.14-5.21`、`5/14~5/21`）。两者中较早的作为起始日（含），较晚的作为截止日（含）。注意区间内的连接符 `-` 与日期内分隔符 `-` 可能混用，按"月.日 到 月.日"的语义判断，例如 `5-14-5-21` 解析为 `5-14` 到 `5-21`。
- **单个日期**：只给一个日期（如 `5.14`）→ 作为起始日，截止日为今天。
- 解析后将"截止日"用于替换下文所有出现的"今天"，将"起始日期"用于 git 的 `--since`。

**1. 确定本周起始日**（仅当用户未指定日期时执行）

目标：与上一份总结严丝合缝衔接；中断/跳周时自动**补齐不留空**。

- 列出 `5.todo/weeks/*.md`，用正则 `(\d{4})-W(\d{2})` 解析文件名里的 ISO 年/周号（只认周号，iCloud 副本如 `2026-W21 2.md` 和 frontmatter 漂移都不会选错文件）。
- **排除本周自己的总结**：跳过周号等于「本周（截止日所在 ISO 周，如 `2026-W23`）」的那一份——它可能是上次生成后被删掉/正在重做的本周总结，不能当作上一份来读。
- 在剩下的里**按 (年, 周) 取最大的一份**作为「上一份总结」，**起始日 = 它覆盖的截止日 + 1 天（含）**：
  - 优先读其 frontmatter 的 `until:` 字段（= 上次复盘真正覆盖到的最后一天）。
  - 若没有 `until:`（旧文件），回退按文件名周号算：起始日 = 该 ISO 周「周日」的次日。
- 一份都没有 → 默认：今天往前 7 天。

> **中断处理（补齐不留空）**：起始日只取决于「上次覆盖到哪天」，与隔了几周无关。若中间跳过若干周，本次窗口会自然横跨这些周、把漏掉的内容一并纳入，不留空档（不设上限）。


**2. 读取每日记录（主叙事来源）**
- 读取 `5.todo/date/` 中从起始日到截止日（未指定日期时即今天）的所有日记文件
- vault 根路径：`<库根目录>`（请替换为你自己的库的绝对路径）

**3. 获取 git 时间线**
```
git -C "<库根目录>" \
  log --name-only --since="[起始日期]" --until="[截止日期]" --format="%ad %s" --date=short
```
（未指定日期时省略 `--until`，即截止到今天）

**4. 获取行为证据（按目录聚合）**
```
git -C "<库根目录>" \
  diff --stat HEAD~30 HEAD -- .
```

**5. 生成并保存**
- 按稳定框架生成完整周复盘
- 在文件末尾追加 ---UPDATE--- 块（格式见稳定框架末尾）
- 保存到：`5.todo/weeks/YYYY-WXX.md`（ISO 周数，如 2026-W21）
- frontmatter 中写入 `date: [起始日期]` 和 `until: [截止日期]`（`until` = 本次真正覆盖到的最后一天，供下次第 1 步衔接，务必如实写）
- 不要询问确认，直接执行并保存

**6. 更新成长档案**
复盘文件保存后，运行：
```
python3 "5.todo/weeks/tools/update-growth.py"
```
